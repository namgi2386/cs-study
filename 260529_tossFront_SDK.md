# 쓰기 쉬운 Toss Front SDK — 정리

> 원문: [쓰기 쉬운 Toss Front SDK](https://toss.tech/article/toss-front-sdk)  
> 저자: 이주함 (토스플레이스 Frontend Developer)

## 핵심 메시지

SDK의 품질은 어떤 기능을 제공하느냐가 아니라, **사용자가 그 기능을 어떤 형태로 쓰게 되느냐**로 결정된다.

---

## 글이 말하는 것

Toss Front(결제 단말기) 위에서 동작하는 플러그인 앱을 외부 개발자가 만들 수 있도록 SDK를 공개했다. 문제는 연동사의 실수가 단말기에서 발생하고, 그 에러는 곧 토스의 이미지 문제가 된다는 것이다.

해결 방향은 **Facade 패턴**이다. 복잡한 내부 절차를 숨기고, 사용자의 의도(Intent) 기준으로 인터페이스를 재구성하는 것. 기능 은닉이 목적이 아니라, 인지 부하를 줄이고 결합도를 낮추는 것이 목적이다.

구체적으로는 두 레이어를 함께 제공한다:

- **High-level(Facade)**: 80% 공통 유즈케이스를 한 번에 처리
- **Low-level(Escape Hatch)**: 세밀한 제어가 필요한 20%를 위한 탈출구

---

## 기술적으로 특별하지 않은 부분

Facade 패턴 자체는 오래된 개념이다. AWS CDK의 L1/L2 구조, React 컴포넌트의 Flat/Compound API 등 이미 업계 전반에서 같은 방향으로 설계하고 있다.

---

## 읽을 만한 이유

**"연동사의 휴먼에러도 우리 책임"이라는 관점이 인상적이다.**

일반적인 SDK 제공사라면 굳이 연동사 실수까지 책임질 이유가 없다. 하지만 Toss Front는 기기가 곧 브랜드다. 플러그인 앱에서 에러가 나도 고객은 토스가 이상하다고 인식한다. 에러의 발생 위치가 책임의 귀속을 결정하는 구조이기 때문에, 연동사 휴먼에러 방지는 선택이 아니라 필수가 된다. 이미지 방어뿐 아니라 SDK팀 자신의 운영 비용을 줄이는 방법이기도 하다.

**토스 TDS 글과 연결되는 지점이 있다.**

TDS 글(Flat/Compound)과 이 글은 출발점이 반대다. TDS는 "통제가 강하면 팀들이 우회한다"는 문제에서, 이 글은 "통제가 느슨하면 연동사가 실수한다"는 문제에서 출발한다. 그런데 도달하는 해법은 동일하다. "쉬운 것은 쉽게, 어려운 것은 가능하게."

"내가 만든 것을 남이 쓴다"는 상황에서 생기는 고민은 결국 수렴한다. SDK든, 디자인 시스템이든, 공통 유틸 함수 하나든 같은 질문이 반복된다. 패턴을 외우는 것보다 이 고민의 본질을 이해하면, 낯선 맥락에서도 같은 원칙을 꺼내 쓸 수 있다.

---

## DX를 고민할 때 항상 같은 원칙이 나오는 이유

이 글과 TDS 글을 함께 읽으면서 든 생각은, DX(Developer Experience)를 진지하게 고민하면 도메인이 달라도 항상 비슷한 결론에 도달한다는 것이다.

SDK 설계든, 컴포넌트 API 설계든, 팀 내 공통 유틸 함수든 — "내가 만든 것을 남이 쓴다"는 구조가 생기는 순간 동일한 질문이 따라온다.

- 사용자가 올바르게 쓰도록 유도할 수 있는가
- 잘못 쓰는 걸 구조적으로 막을 수 있는가
- 단순한 케이스는 단순하게, 복잡한 케이스는 가능하게 할 수 있는가

이 질문들은 특정 패턴에서 비롯된 게 아니다. 오히려 반대다. 이 질문들에 답하다 보면 자연스럽게 Facade, Compound, Escape Hatch 같은 패턴에 도달하게 된다. 패턴이 먼저가 아니라, 고민이 먼저다.

그래서 Facade 패턴을 "이럴 때 쓰는 패턴"으로 외우는 것과, "왜 이 패턴이 만들어졌는가"를 이해하는 것은 실용적인 차이가 있다. 전자는 같은 상황에서만 꺼낼 수 있고, 후자는 전혀 다른 맥락에서도 같은 사고방식을 적용할 수 있다.

개발자로서 특정 기술이나 패턴을 익히는 것보다, **"사용자가 무엇을 하려는가"를 먼저 묻는 습관**이 더 오래가는 자산이라는 생각이 들었다.

---

## 한 줄 정의

> 플랫폼 위에서 남이 만든 결과물도 내 얼굴이 된다는 전제 위에서, 사용자의 실수를 구조적으로 방지하는 SDK 설계 원칙을 다룬 글.
